🐛 Bugs encontrados e corrigidos

1
PATCH de layout sobrescrevia type e config
server/src/features/dashboardLayouts/services/DashboardLayoutService.ts
O merge parcial lia layout.layoutData, que não existe em IDashboardLayout. O PATCH retornava HTTP 200 mas perdia os campos type e config, corrompendo o registro no banco. Qualquer edição parcial de nome destruía silenciosamente a configuração do layout.
- name: data.name ?? layout.layoutData?.name - type: data.type ?? layout.layoutData?.type + name: data.name ?? layout.name + type: data.type ?? layout.type + config: data.config ?? layout.config
2
Rota estática /qdrant-status capturada por /:id
server/src/routes/documents.ts
Rotas estáticas como GET /qdrant-status, POST /search e GET /list foram declaradas após GET /:id no router do Express. O parâmetro dinâmico as interceptava, retornando 404 ou resultado incorreto para qualquer chamada que não fosse busca por ID.
- router.get('/:id', getDocumentById) // rota dinâmica ANTES - router.get('/qdrant-status', getQdrantStatus) // nunca alcançada - router.post('/search', searchDocuments) // nunca alcançada + router.get('/qdrant-status', getQdrantStatus) // rotas estáticas PRIMEIRO + router.post('/search', searchDocuments) + router.get('/list', listDocumentNames) + router.get('/:id', getDocumentById) // dinâmica por último
3
POST /chat-messages retornava 500 — 'USER' vs 'user'
server/src/features/chatMessages/services/ChatMessageService.ts
O service hardcodava role: 'USER' (maiúsculo) para hardforçar segurança. Porém o repositório fazia switch em ChatMessageRole.USER = 'user' (minúsculo). Nenhum case do switch batia → convertDomainRoleToPrismaRole retornava undefined → erro de constraint no Prisma. Toda criação de mensagem de chat falhava com HTTP 500.
- role: 'USER' as unknown as MessageRole // maiúsculo: switch não bate + role: 'user' as unknown as MessageRole // minúsculo: matches ChatMessageRole.USER

⚠️ Problemas conhecidos (não críticos)

  • Qdrant offline: busca semântica de documentos indisponível em dev local. Instalar o Qdrant Docker ou apontar para instância cloud para habilitar POST /documents/search.
  • PDF text extraction falha (Command token too long): LibreOffice / pdftotext não instalado no ambiente. Upload retorna 202 (async), mas o processamento falha imediatamente com FAILED.
  • GET /api/users/me retorna 404 — rota inexistente no router de users. A rota correta para perfil autenticado é /api/auth/me.
  • analyticsDefinitions CORE table não encontrada no seed atual — POST /analytics/definitions inoperante (requer tabela criada via preset ou seed específico).
  • revenueByChannel parcial: canais Telefone, Outros, Loja física e App retornam zero por ausência de dados semeados — comportamento esperado e documentado.
  • sidebar "commercial" retorna count:0 — nenhum dado na categoria commercial no seed atual. Não afeta funcionalidade.
  • Chat instance sem GET /:id — a rota de busca por ID de instância não existe; o frontend usa a lista completa para localizar instâncias.
